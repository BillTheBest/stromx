Develop

To play around with stromx and implement first vision streams an installation of stromx-studio is the only thing you need. It can be downloaded for the most popular Linux distributions and for Windows on the downloads page.

It is also possible to use the functionality of stromx from a different application via the stromx API. E.g. a vision application with a custom GUI can use stromx to implement its imaging capabilities. In this case the stromx header files and the shared library objects are necessary. The same files – headers and shared objects – are necessary if you want to implement your own operators. For those Linux distributions stromx is packaged for these files are provided in the devel packages. For other distributions stromx has to be compiled from the sources to be linked against.

On Windows the developer files of the stromx library have to be build for each version of Visual Studio. For this reason we currently do not provide pre-compiled Windows versions. Finally, if you want to participate in the development of stromx or to adapt the library to your needs it is obviously necessary to compile and build it yourself. The necessary information on how to build stromx is given below.

Sources

The sources of stromx are hosted at github. For stromx-studio a separate repository exists here. You can either clone the repository with git or download the repository as a compressed archive.

Build instructions

The common wiki of both projects provides useful information about the necessary prequisites for building stromx and stromx-studio on Linux and Windows. It also contains build instruction for both platforms.

Documentation and tutorials

The documentation of the stromx API is generated during the build process as explained in the wiki. Alternatively, a copy of the documentation is available online. It includes tutorials which explain how to use stromx in an application and how to implement custom operator.
